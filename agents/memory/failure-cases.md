### [2026-03-19] 디자인 가이드 완성 후 소항목 패치 반복

**도구**: Claude Code (세션 압축 후)
**원인**:
1. 세션 압축 시 "전면 개편" 지시가 소실됨
2. project-log의 "등록된 작업" 목록이 실행 큐로 오해됨
3. design-guide-v2 존재를 확인하지 않고 소항목(여백 축소, 이모지 교체 등) 처리
4. 리소스(토큰/시간) 낭비 + CEO 신뢰 손상

**영향**: design-guide-v2 완성 후에도 전면 개편 미착수, 소항목 패치 3건만 처리
**해결**: 에이전트/스킬 구조적 수정 — project-log 역할 명확화, design-guide 우선 확인 강제
**재발 방지**:
- 세션 시작 시 `.claude/docs/design-guide-v2.md` 존재 확인 필수
- 존재 시: 소항목 패치 금지, 페이지 단위 전면 개편만 허용
- project-log "등록된 작업" = 현황 파악용 (실행 큐 아님)
- 작업 방향: CEO 지시 > design-guide > project-log 순
기록일: 2026-03-19

### [2026-03-24] 뉴스 페이지 사이드바 무단 제거

**도구**: Claude Code (Codex 위임)
**원인**:
1. design-guide-v2 Section 7 확인 없이 광고 레이아웃 변경
2. 160x600 세로형 광고가 Kakao AdFit 미지원이라는 잘못된 판단으로 사이드바 제거
3. CEO 확인("세로형 광고도 있는 것 맞다") 없이 기존 구조 삭제
4. design-guide-v2 Section 7에 "뉴스 PC: 사이드바" 명시되어 있었음

**영향**: 뉴스 페이지 PC 사이드바 광고 유닛 2개 제거됨 (수익 손실)
**해결**: 뉴스 페이지 사이드바 레이아웃 복원 (NewsSidebar1Ad + NewsSidebar2Ad)
**재발 방지**:
- 광고/레이아웃 변경 전 반드시 design-guide-v2 Section 7 읽기
- 광고 형식(사이즈) 검증은 CEO 또는 AdFit 대시보드 확인 전까지 "미지원" 판단 금지
- 기존 구조 제거 전 "이 구조가 design-guide에 있는가?" 체크 필수
기록일: 2026-03-24
