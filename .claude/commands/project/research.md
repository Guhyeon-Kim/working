# 리서치 요청

사용자의 리서치 요청: $ARGUMENTS

## 실행 순서

1. **주제 분석**: 리서치 목표, 범위, 핵심 질문 정리
2. **research-agent 위임**: Gemini CLI를 통한 멀티소스 조사
   - 웹 검색 (최신 트렌드, 시장 데이터)
   - GitHub 검색 (오픈소스 사례, 기술 벤치마킹)
   - 학술/전문 자료 (해당 시)
3. **검증 및 가공**: 
   - 출처 신뢰도 평가 (✅ 검증됨 / ⚠️ 미검증)
   - 핵심 인사이트 추출
   - 우리 상황에 적용 가능한 시사점 도출
4. **결과 정리**: 구조화된 리서치 보고서 작성

## 출력 형식

```markdown
## 리서치: [주제]
### 핵심 발견
### 상세 분석
### 시사점 및 적용 방안
### 출처
```

## 규칙

- 추측이 아닌 실제 데이터만 인용
- 출처 URL 필수 포함
- 우리 프로젝트와의 관련성을 명시적으로 연결
