# 산출물 리뷰 요청

리뷰 대상: $ARGUMENTS

## 실행 순서

1. **산출물 식별**: 리뷰 대상 파일/디렉토리 확인
2. **정합성 검사**: 
   - 요건명세 ↔ 와이어프레임: 정의된 필드가 화면에 반영되었는가
   - 와이어프레임 ↔ 플로우차트: 화면 전환이 플로우와 일치하는가
   - 요건명세 ↔ 테스트 시나리오: 모든 요건이 테스트로 커버되는가
   - 예외 케이스: 에러/빈 상태/부분 실패가 모두 정의되었는가
3. **품질 체크리스트**:
   - [ ] 기능 요구사항 완전성
   - [ ] 비기능 요구사항 (성능, 보안, 접근성)
   - [ ] 엣지 케이스 커버리지
   - [ ] 용어 일관성
   - [ ] 구현 가능성 (기술적 feasibility)
4. **리뷰 보고**: `/review-request` 스킬 형식으로 결과 보고

## 출력 형식

```markdown
## 리뷰 결과: [대상]

### PASS 항목
### WARN 항목 (개선 권장)
### BLOCK 항목 (수정 필수)
### 정합성 매트릭스
```
