# 새 프로젝트 기획 시작

사용자의 기획 요청: $ARGUMENTS

## 실행 순서

1. **요청 분석**: 사용자 요청에서 핵심 목표, 대상 사용자, 제약 조건을 추출
2. **리서치**: research-agent를 통해 경쟁사/시장/기술 벤치마킹 수행
3. **요건 정의**: `/planning` 스킬로 기능/비기능 요구사항, 우선순위, 필수/선택 여부 정의
4. **와이어프레임**: `/wireframe` 스킬로 화면 구성, 컴포넌트 배치 정의
5. **플로우차트**: `/flowchart` 스킬로 사용자 시나리오, 분기, 오류 흐름 정의
6. **산출물 정합성 검증**: 요건 ↔ 와이어프레임 ↔ 플로우 간 필드/예외 일치 확인
7. **CEO 컨펌 요청**: `/review-request` 스킬로 산출물 요약 보고

## 산출물

- requirements.md (요건 명세)
- wireframe.md (와이어프레임)
- flowchart.md (서비스 플로우)
- test-scenarios.md (테스트 시나리오)

## 규칙

- 산출물은 프로젝트 디렉토리에 저장 (없으면 생성)
- 각 산출물 간 정합성 필수 검증
- CEO 컨펌 전까지 다음 단계 진행 금지
