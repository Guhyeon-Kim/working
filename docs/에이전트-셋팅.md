```
[불변 규칙] Codex/Gemini CLI 직접 호출 금지.

  node ~/.claude/scripts/delegate.mjs codex frontend "로그인 페이지 구현"
  node ~/.claude/scripts/delegate.mjs codex backend "stocks API 구현"
  node ~/.claude/scripts/delegate.mjs gemini research "경쟁사 분석"
  node ~/.claude/scripts/delegate.mjs gemini design "대시보드 화면 초안"

delegate.mjs가 자동으로 하는 일:
  1. 프로젝트 컨텍스트에서 관련 파일 탐색 (api-spec, design-spec 등)
  2. 작업 유형별 컨텍스트 패킷 조립
  3. 반복 버그 레지스트리 + 코딩 규칙 자동 포함
  4. 패킷을 .claude/delegation/에 감사용 저장
  5. CLI 호출 + 결과 반환

enforce-delegation.mjs (v2.0)가 직접 호출을 차단하고 delegate.mjs 사용을 강제한다.
```

## 8. 자기 성장 시스템 (v6.1)

### 전체 흐름

```
세션 시작
  └── session-start-memory.mjs (v6.1)
        ├── turn-counter.json 리셋 (Context Rot 방지) [NEW]
        ├── evolving-rules.json 로딩 (반복 에러 경고)
        ├── failure-cases.md 요약 (장애 사례 수)
        ├── success-patterns.md 요약 (성공 패턴 수)
        ├── acl-state.json 확인 (미해결 빌드 에러)
        ├── cli-status.json 확인 (CLI 불가 상태)
        ├── pending-hookify.json → hookify 자동 트리거 프롬프트 [NEW]
        └── health-dashboard.md 자동 생성 [NEW]

세션 중
  ├── on-prompt-unified.mjs → CLI 체크 + 품질 GC + 위험 감지 + Context Rot 방지 (4개 통합)
  ├── enforce-delegation.mjs (v2.1) → delegate.mjs bypass + 코드 10줄 임계값
  ├── delegate.mjs → 컨텍스트 패킷 + 결과 품질 피드백 로그 [NEW]
  ├── acl-post-build.mjs → acl-state.json 기록
  └── acl-subagent-stop.mjs (v2.0) → 빌드 캐시 + execFileSync [NEW]

세션 종료
  ├── on-stop.mjs (v4.0) → Node.js 전환, 인코딩 안전 [NEW]
  └── evolving-guardrails.mjs (v5.3)
        ├── acl-state.json에서 실패 수집
        ├── quality-gate-log.md에서 실패 수집
        ├── activity-log.md에서 재시작 패턴 수집
        ├── evolving-rules.json에 패턴 축적
        ├── pending-hookify.json 자동 생성 (count >= 3) [NEW]
        │     count >= 2 → hookify 권장
        │     count >= 3 → pending-hookify.json 등재 + 다음 세션 자동 프롬프트
        │     count >= 5 → failure-cases.md + CLAUDE.md 등재 권고
        └──── 30일+ 미발생 → 자동 아카이브 (GC)
```

### 학습 루프 (v6.0 — 자동화 강화)

```
실패 발생 → evolving-rules.json 축적
         → 다음 세션 시작 시 경고 주입
         → 같은 실수 사전 차단
         → count >= 3: pending-hookify.json 자동 등재 [NEW]
         → 다음 세션: hookify 실행 강하게 프롬프트 [NEW]
         → hookify 규칙 생성 → PreToolUse 자동 차단
         → count >= 5: failure-cases.md에 정식 등재
         → 30일 미발생: 자동 아카이브 (해결 간주)
```
