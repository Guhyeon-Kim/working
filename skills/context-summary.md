# context-summary

**버전**: v6.0
**주 사용 에이전트**: curator, researcher
**연계 스킬**: project-log, onboarding

---

## 목적

대량의 컨텍스트(긴 대화·여러 문서·다수 로그)를 **정보 손실 최소화**하면서
**탐색 가능한 형태**로 요약. 미래의 자신·AI 세션이 원본 복원 가능하게.

---

## 호출 시점

- 긴 세션 중간에 컨텍스트 창 정리 필요 시
- 휴식·다음 날 재개 전 "오늘 여기까지" 스냅샷
- 여러 세션·문서에 걸친 주제 통합
- 리서치 결과의 executive summary
- 프로젝트 상태 핸드오프 (외부 협업자·새 AI)

---

## 입력

- 요약 대상 (대화 로그·문서·코드 변경 등)
- 독자 (자신 / CEO / 새 AI 세션 / 외부)
- 목적 (재개용 / 보고용 / 참조용)
- 분량 제약 (있으면)

---

## 절차

### 1. 독자·목적 확정
- **재개용**: 작업의 다음 스텝이 명확해야 함
- **보고용**: 결론·결정사항이 선명해야 함
- **참조용**: 탐색성 (링크·인덱스) 높아야 함

### 2. 정보 계층 분리

**반드시 보존**:
- 결정사항 (무엇을 왜 결정했나)
- 미결 사항 (해결 안 된 질문·이슈)
- 다음 단계 (무엇을 해야 하는가)
- 링크·참조 (원본 접근 경로)

**요약 가능**:
- 논의 과정 (결론만 남기고 과정은 압축)
- 대안 비교 (선택된 것만 상세, 나머지는 이름)

**생략 가능**:
- 인사·사적 대화
- 중복 설명
- 폐기된 방향

### 3. 구조 선택
목적별로:

- **재개용**: 시간 축 (무엇을 했고 다음엔 뭐)
- **보고용**: 결론 우선 (TL;DR → 상세)
- **참조용**: 주제별 (카테고리 인덱스)

### 4. 초안 작성
- TL;DR 먼저 (3줄 이내)
- 본문은 계층 따라
- 원본 링크 필수

### 5. 검증
- 원본 없이 이 요약만 보고 이해 가능한가
- 링크 유효한가
- 결정·미결·다음 단계 빠짐없는가

## 요약 기법

### 압축 비율 기준
- 10:1 — 대화 로그 → 스냅샷
- 30:1 — 주간 리포트
- 100:1 — 월간 executive summary

### 정보 손실 최소화 기법
1. **원본 링크 유지**: 상세 필요 시 원본 접근 가능
2. **구체성 보존**: 숫자·고유명사·날짜는 요약해도 유지
3. **결정의 근거**: "무엇"과 함께 "왜"
4. **반대 관점 기록**: 폐기된 대안도 한 줄 남김 (왜 안 됐나)

### 표현 효율화
- "논의 결과 A로 결정" → "A 결정" (과정 생략)
- "여러 후보 검토 후 B 선택" → "B 선택 (대안: C, D, 이유: ...)"
- "여러 번 시도했으나" → "N회 시도 후"

## 체크리스트

- [ ] TL;DR 3줄 이내
- [ ] 결정사항 모두 포함
- [ ] 미결 사항 명시
- [ ] 다음 단계 구체적 (독자가 행동 가능)
- [ ] 원본 링크 유효
- [ ] 구체 정보(숫자·날짜) 보존
- [ ] 목적·독자 일치하는 구조

---

## 금지

- **링크 없는 요약**: 원본 복원 불가
- **결정만 남기고 근거 생략**: "왜"가 빠지면 미래에 재논의 필요
- **과도한 축약**: 일반화된 문장으로 구체성 사라짐
- **자의적 삭제**: 요약자가 "중요하지 않다" 판단한 정보 삭제 (최소 언급)
- **독자 무시**: 재개용인데 보고 포맷 사용 (다음 단계 없음)
