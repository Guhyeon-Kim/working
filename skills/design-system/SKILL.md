# Design System 스킬

## 핵심 원칙

> 디자인 시스템은 "예쁜 색상 모음"이 아니다. 제품의 일관성을 보장하는 계약서다.
> 디자인 시스템 없이 UI를 만드는 것은 API 명세 없이 백엔드를 만드는 것과 같다.
> 모든 디자인 판단은 이 문서에 근거해야 한다. 근거 없는 디자인 판단은 기술 부채다.

---

## 트리거 조건

```
다음 중 하나에 해당하면 이 스킬이 자동 로드된다:
1. 신규 프로젝트 온보딩 시 (.claude/docs/design-system.md 미존재)
2. 사용자가 "디자인 시스템", "디자인 가이드", "스타일 가이드" 작성을 요청
3. design-agent가 호출됐는데 design-system.md가 없을 때 (BLOCKING)
```

## Step 1: 프로젝트 정보 수집

사용자에게 다음 정보를 확인한다. 누락 시 반드시 질문한다.

```
필수 정보:
□ 프로젝트명 / 서비스명
□ 서비스 유형 (웹앱 / 모바일앱 / 대시보드 / 랜딩 / 블로그 등)
□ 대상 사용자 (연령대, 기술 수준, 사용 환경)
□ 브랜드 톤 (전문적 / 친근 / 미니멀 / 대담 / 고급 등)
□ 레퍼런스 서비스 (참고할 서비스 2~3개)

선택 정보 (있으면 더 정확):
□ 브랜드 컬러 (이미 정해진 경우)
□ 로고 / 브랜드 에셋
□ 기존 디자인 (리디자인인 경우)
□ 특수 요구사항 (접근성 수준, 다크모드, 다국어 등)
□ 금융/의료/교육 등 규제 산업 여부
```

## 5. 컴포넌트 기준

### UI 라이브러리
- 기본: shadcn/ui (커스텀보다 shadcn 우선)
- 아이콘: Lucide React
- 차트: Recharts (dynamic import 필수)
- 애니메이션: Tailwind transition (복잡한 경우만 Framer Motion)

### 컴포넌트 우선순위
```
1. shadcn/ui 기본 컴포넌트 사용
2. shadcn 조합으로 Composition
3. 커스텀 구현 (위 두 가지 불가 시만)
```

### 5-State 필수 (모든 데이터 컴포넌트)
| 상태 | 표시 | 컴포넌트 |
|------|------|---------|
| 기본 | 데이터 정상 표시 | - |
| 로딩 | Skeleton (레이아웃 유지) | Skeleton |
| 빈 상태 | 안내 문구 + CTA | EmptyState |
| 에러 | 에러 메시지 + 재시도 | Alert + Button |
| 부분 실패 | 성공 부분 표시 + 실패 안내 | Partial |

### 버튼 체계
| variant | 용도 | 화면당 개수 |
|---------|------|------------|
| default (primary) | 핵심 CTA | 최대 1개 |
| secondary | 보조 액션 | 자유 |
| outline | 취소, 덜 중요한 액션 | 자유 |
| ghost | 메뉴, 최소 강조 | 자유 |
| destructive | 삭제, 위험 액션 | 확인 다이얼로그 필수 |

## 6. 접근성 (Accessibility)

### 기준
- WCAG 2.1 AA (최소)
- 색상 대비: 4.5:1 이상 (본문), 3:1 이상 (대형 텍스트)

### 필수 체크리스트
```
□ 인터랙티브 요소에 aria-label
□ 로딩: aria-busy="true"
□ 에러: role="alert"
□ 모달: focus trap + ESC 닫기
□ 색상만으로 상태 구분 금지 (아이콘/텍스트 병행)
□ 키보드 Tab 순서 논리적
□ 이미지: alt 텍스트 필수
```

## 7. 산업별 추가 규칙 (해당 시)

### 금융 서비스
- 수익/손실 색상 구분 필수 (--success / --danger)
- 숫자 표시: tabular-nums + 천 단위 구분자
- 면책 고지 필수 (추천/전략 기능)
- WHY + RISK + CONDITION/EXIT 구조

### 의료/헬스
- 민감 정보 마스킹 기본
- 접근성 AAA 권장

### 교육
- 읽기 영역 narrow 기본
- 단계별 진행 표시 필수

## 컬러 생성 가이드라인

사용자가 브랜드 컬러만 제공한 경우, 나머지 팔레트를 파생한다:

```
1. Primary → Primary-hover: 10% 어둡게
2. Primary → Ring: 20% 투명도
3. BG: Primary 계열 극저채도 (채도 5~15%, 명도 95~98%)
4. Surface: BG보다 약간 밝게 (보통 #FFFFFF)
5. Text: BG와 대비 7:1 이상
6. Text-secondary: 대비 4.5:1
7. Text-muted: 대비 3:1 (장식적 텍스트만)
8. Border: Text 대비 20~30% 투명도
9. Success: 녹색 계열 (채도 높음)
10. Danger: 빨강 계열 (채도 높음)
11. Warning: 노랑/주황 계열
```

## 작업 원칙

```
□ 프로젝트 정보 없이 임의 생성 금지 — 반드시 정보 수집 후 진행
□ 브랜드 컬러 미정 시 사용자에게 확인 (임의 선정 금지)
□ 레퍼런스 없으면 서비스 유형 기반 기본 레퍼런스 제안
□ draft 상태로 생성 → 사용자 confirmed 후에만 design-agent 진행 가능
□ 이 문서가 모든 디자인 판단의 단일 소스 (Single Source of Truth)
```
