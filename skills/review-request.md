# review-request

**버전**: v6.0
**주 사용 에이전트**: builder, tester, designer, planner
**연계 스킬**: self-review

---

## 목적

CEO·CTO에게 리뷰·승인 요청 시 **맥락 복원 비용을 최소화**한 포맷. 받는 사람이
어떤 판단을 내려야 하는지 명확.

---

## 호출 시점

- CEO 컨펌 게이트 (planner 요건서 승인)
- CTO 리뷰 (builder 코드, designer 안, tester 리포트)
- 배포 승인 요청 (tester PASS 후)
- 범위·방향 결정 요청 (선택지 2~3개 제시)

---

## 입력

- 산출물
- 요청 유형 (컨펌 / 리뷰 / 선택 / 결정)
- 기한 (있으면)

---

## 절차

### 1. 요청 유형 명확화
- **컨펌**: Yes/No 결정 요청
- **리뷰**: 피드백 수집 (수정 여지 있음)
- **선택**: 2~3개 안 중 고르기
- **결정**: 방향 결정 (ex. 이 기능 할 것인가 말 것인가)

### 2. TL;DR 먼저
최대 3줄:
- 한 줄: 무엇에 대한 요청인가
- 두 줄째: 핵심 포인트
- 세 줄째: 무엇을 해주면 되는가 (ex. "승인해주세요")

### 3. 본문 구조
- 배경 (2~3문장)
- 산출물 요약 (링크 + 핵심 3~5포인트)
- 결정 사항 (이미 정한 것)
- 질문·대안 (결정해달라는 것)
- 리스크 (있으면)

### 4. 응답 편의성
- 승인 옵션 제시 (이 형태로 답변해주세요)
- 기한 명시 (가능한 경우)
- 추가 정보 위치 (링크)

### 5. 형식
- 채팅: 짧게, 3~10줄
- Notion: 중간, 구조화
- 파일: 상세, 정식 문서

---

## 출력

### 짧은 채팅 포맷 (컨펌 요청)

```
[컨펌 요청] 허브와이즈 편향지수 API 요건서 v1.0

핵심:
- 편향 4종 계산 (FOMO, 군중, 손실회피, 과신)
- 매매 5건 미만은 204 No Content
- Rate limit 10회/일 (사용자당)

결정 필요:
- 위 요건으로 builder에 전달해도 될까요?

승인 방식: "OK" 또는 수정 요청 (어떤 부분)
참고: <요건서 URL>
```

### 선택 요청 (2~3안 제시)

```
[선택 요청] 편향 지수 계산 주기

옵션 A: 사용자 요청 시 실시간
  + 최신 데이터
  - 호출마다 계산 (비용·지연)

옵션 B: 매일 새벽 일괄 계산
  + 응답 빠름
  - 데이터 지연 최대 24시간

옵션 C: 하루 1회 캐시 + 사용자 요청 시 갱신 가능
  + 절충
  - 구현 복잡

기한: 내일까지 (월요일 구현 시작)
추천: B (MVP로 충분)
어떻게 갈까요?
```

### 배포 승인 요청

```
[배포 승인] 편향 지수 API 프로덕션 배포

- builder 완료
- tester PASS (8/8 케이스)
- 성능: p95 2.8초 (기준 3초 이내)
- 링크: PR #123

승인하시면 즉시 배포합니다.
```

### Notion 페이지 구조 (상세 리뷰)

```markdown
# [리뷰 요청] <산출물명>

**요청자**: <역할>
**요청일**: YYYY-MM-DD
**유형**: 리뷰
**기한**: YYYY-MM-DD (있으면)

---

## TL;DR
- <1줄>
- <1줄>
- <1줄: 무엇을 해주면 되는가>

## 배경
<2~3문장>

## 산출물
- 파일:
- 주요 변경:
- 링크:

## 이미 정한 것
- <결정 1>
- <결정 2>

## 리뷰받고 싶은 부분
- <구체적 질문 1>
- <구체적 질문 2>

## 리스크
- <있으면>

## 응답 편의
- 이 페이지 댓글로 피드백 남겨주세요
- 또는 Slack DM
```

## 체크리스트

- [ ] TL;DR 있음
- [ ] 요청 유형 명시 (컨펌/리뷰/선택/결정)
- [ ] 기한 명시 (가능한 경우)
- [ ] 응답 방법 안내
- [ ] 참고 링크 포함
- [ ] 이미 정한 것과 열린 질문 구분

---

## 금지

- **맥락 없는 질문 폭탄**: "이거 어떻게 할까요?" 맥락 누락
- **결정 전가**: "어떻게 할지 알려주세요" → 자신의 추천·근거 먼저
- **긴 서문**: 본론 전에 배경 10줄 → TL;DR로 시작
- **응답 양식 불명**: "의견 주세요" → 구체적 응답 포맷 제시
