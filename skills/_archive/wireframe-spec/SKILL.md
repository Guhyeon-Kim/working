# Wireframe Spec

디자이너가 디자인하고, 개발자가 구현할 수 있는 wireframe 작성 방법.

## 좋은 Wireframe의 기준

- 각 화면의 **목적**이 명확한가
- 표시할 **데이터**가 명시됐는가
- 사용자 **행동**과 **결과**가 정의됐는가
- **상태** (로딩/에러/빈 데이터)가 모두 정의됐는가

## 화면당 필수 요소

### 1. 레이아웃 구조
어떤 영역이 어디에 있는지 텍스트로 표현:
```
상단 헤더: 페이지 제목 + 우측 액션 버튼
중앙 본문: 카드 그리드 (3열, 모바일 1열)
하단: 페이지네이션
```

### 2. 컴포넌트 목록과 데이터
```
- 종목 카드: 종목명, 현재가, 등락률, 52주 범위
- 검색바: 플레이스홀더 "종목명 또는 코드 검색"
- 필터 버튼: 전체 / 코스피 / 코스닥 / ETF
```

### 3. 인터랙션
```
- 카드 클릭 → /stocks/[code]로 이동
- 검색어 입력 (300ms debounce) → 결과 실시간 필터
- 필터 선택 → URL 쿼리 파라미터로 유지
```

### 4. 상태 처리 (4개 모두 필수)
```
로딩: 카드 3개 skeleton (애니메이션)
에러: "데이터를 불러오지 못했습니다" + 재시도 버튼
빈 데이터: "검색 결과가 없습니다" + 검색어 초기화 버튼
기본: 카드 목록 표시
```

## HubWise 레이아웃 패턴

### 목록 페이지
```
페이지 헤더 (제목 + 설명)
필터/검색 영역
콘텐츠 (카드 그리드 또는 테이블)
페이지네이션
```

### 상세 페이지
```
브레드크럼 + 뒤로가기
헤더 (핵심 정보 요약)
탭 (여러 섹션이 있을 경우)
본문 콘텐츠
관련 액션 버튼
```

### 폼 페이지
```
페이지 제목
입력 필드들 (라벨 + 입력 + 힌트)
유효성 에러 (필드 아래)
제출 / 취소 버튼
```
