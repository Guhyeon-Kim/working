# wireframe

**버전**: v6.0
**주 사용 에이전트**: designer
**연계 스킬**: wireframe-spec, flowchart, ui-component

---

## 목적

UI 화면의 정보 구조·레이아웃·우선순위를 시각화. 픽셀 완성도보다 "무엇이
어디에 왜 있는가"에 집중.

---

## 호출 시점

- 새 페이지·모달·섹션 신설
- 기존 UI 정보 구조 재설계
- designer가 와이어프레임 2~3개 후보 제안 시

---

## 입력

- 요건서 (requirements-spec 산출물)
- 대상 화면 URL/경로
- 디바이스 우선순위 (모바일/데스크톱)
- 기존 컴포넌트·디자인 시스템 참조

---

## 절차

### 1. 정보 위계 정하기
사용자가 이 화면에 왔을 때 알아야 할 순서:
1. 가장 먼저
2. 그 다음
3. 부가 정보
→ 시각적 위계(크기·위치)가 이 순서를 반영하는가

### 2. 주요 영역 블록 정의
- 헤더 (네비게이션·로고)
- 히어로 / 메인 컨텐츠
- 보조 컨텐츠 (사이드바 등)
- 푸터
- 모달·오버레이 (있으면)

### 3. 반응형 고려
- 모바일 (≤640px): 단일 컬럼, CTA 풀너비, 터치 타깃 44px 이상
- 태블릿 (641~1024px): 2 컬럼 가능
- 데스크톱 (≥1025px): 최대 폭 제한, 여백 활용

### 4. 상태 그리기
- 기본 (default)
- 호버·포커스
- 로딩
- 빈 상태 (데이터 없음)
- 에러

### 5. 인터랙션 표기
- 클릭 시 어디로 이동
- 호버 시 어떤 변화
- 애니메이션 (필요 시)

### 6. 컴포넌트 매핑
각 블록 → shadcn/ui 또는 기존 컴포넌트 / 신규 필요 여부

## 출력

ASCII 또는 Mermaid 형태:

### ASCII 예시
```
┌──────────────────────────────────┐
│  [로고]              [메뉴]      │  ← 헤더
├──────────────────────────────────┤
│                                  │
│    <H1> 메인 헤드라인             │  ← 히어로
│    <p>  서브 메시지               │
│                                  │
│    [주요 CTA]  [보조 CTA]         │
│                                  │
├──────────────────────────────────┤
│  [카드 1] [카드 2]               │  ← 기능
│  [카드 3] [카드 4]               │
└──────────────────────────────────┘
```

### 모바일 버전
```
┌────────────────┐
│ [로고]    [≡] │
├────────────────┤
│                │
│  <H1>          │
│  <p>           │
│                │
│ [주요 CTA]     │
│ [보조 CTA]     │
├────────────────┤
│ [카드 1]       │
│ [카드 2]       │
│ [카드 3]       │
└────────────────┘
```

### 상태 명세
- 로딩: 스켈레톤 (카드 영역 회색 박스)
- 빈 상태: "아직 데이터가 없어요" + CTA
- 에러: 상단 알럿 + 재시도 버튼

## 금지

- **픽셀 조정 제안**: "버튼 8px 패딩" → builder 재량
- **단일 안만 제시**: CTO 선택권 박탈
- **해피 패스만**: 상태 누락
- **디자인 시스템 이탈**: 프로젝트 토큰 외 임의 색·간격
